# 产品最高约束

## 状态

Active

## 核心要求

`we together` 的所有开发必须遵守以下最高优先级要求：

### 1. 严格工程化

项目推进必须严格工程化进行：

- 规划
- 跟进
- 归档
- 约束

四者缺一不可。

### 2. 通用型 Skill

最终产物必须是一个通用型 Skill，而不是绑定单一平台的项目。

架构目标必须支持：

- Claude
- Codex
- 任意 LLM Runtime / Skill Host

### 3. 产品最终形态

最终产品应是一个：

- 自适应的
- 可扩展的
- 具备神经单元网格式激活传播特征的
- 可持续演化的

数字赛博生态圈产品。

## 工程含义

这意味着：

- 核心 schema 不能写死为某个平台的数据格式
- 运行时不能写死为单一宿主 API
- importer 不能只为一个平台服务
- 文档和状态跟踪必须长期维护
