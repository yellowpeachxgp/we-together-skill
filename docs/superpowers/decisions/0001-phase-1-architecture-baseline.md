# ADR 0001：Phase 1 架构基线

## 状态

Accepted

## 背景

`we together` 的目标不是单人物 Skill，而是一个可运行、可演化、可追溯的社会图谱 Skill。

如果不先建立统一架构基线，后续会在以下方向上快速失控：

- 工作面与亲密面形成双系统拼接
- 自动导入与运行时演化使用两套不兼容模型
- 人物、关系、场景、事件、记忆、状态的边界模糊
- 多人互动缺乏运行时约束

## 决策

第一阶段接受以下架构基线：

1. 锚点场景选择 `C：混合小社会`
2. 采用统一社会图谱内核，不做工作系统与亲密系统双拼
3. 默认导入全自动、自动入图谱
4. 身份融合默认激进，但必须可逆、可追溯
5. 演化采用 `Event -> Patch -> Graph State -> Snapshot`
6. 第一阶段只支持局部分支，不支持整图分叉
7. 运行时采用“有界激活传播模型”
8. 环境参数采用“核心维度固定 + 自定义扩展”
9. 主存储采用 SQLite + 文件系统混合模型
10. importer 统一输出为“证据层 + 候选层”对象

## 结果

这使得第一阶段可以在不失去最终方向的前提下，先实现：

- 规范主对象
- 运行时激活逻辑
- 自动导入与身份融合
- 可追溯留痕链

同时避免一开始就掉入：

- 大而全 schema
- 无边界群聊
- 单人物架构拼接
- 纯 prompt 系统

## 后续

后续设计和实现若偏离本 ADR，应新增新的 ADR 说明原因与替代方案。
